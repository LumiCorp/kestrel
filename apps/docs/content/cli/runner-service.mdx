<GuideNote step="Local integration">
Give a trusted local application server an authenticated HTTP connection to Local Core.
</GuideNote>

`kestrel web` exposes Local Core's Execution Protocol through an authenticated
loopback HTTP endpoint. It is a bridge for trusted local server applications,
not a second runtime.

## Start the bridge

```bash
kestrel web
```

The command ensures Local Core is ready and prints:

- `KESTREL_RUNNER_SERVICE_URL`
- `KESTREL_RUNNER_SERVICE_TOKEN`

## Connect a local application

```bash
kestrel web
export KESTREL_RUNNER_SERVICE_URL='http://127.0.0.1:43102'
export KESTREL_RUNNER_SERVICE_TOKEN='...'
pnpm dev
```

The SDK uses its remote HTTP transport for this loopback URL, but execution and
state remain in Local Core.

## Check health and authentication

```bash
curl -sS "$KESTREL_RUNNER_SERVICE_URL/health"
```

If you provided a custom host, port, or token, confirm the printed exports match
the values used by the local application server.
