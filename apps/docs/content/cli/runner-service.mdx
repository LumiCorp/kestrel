This is the terminal-side entrypoint for the same runner service used by `/api/copilot/run` and `/api/copilot/stream`.
</WorkspaceCopilotDemo>

Use these commands when you want to start the local HTTP runner directly from a terminal.

## Start it

```bash
kestrel web
```

The command prints:

- `KESTREL_RUNNER_SERVICE_URL`
- `KESTREL_RUNNER_SERVICE_TOKEN`

## Common local flow

```bash
kestrel web
export KESTREL_RUNNER_SERVICE_URL='http://127.0.0.1:43102'
export KESTREL_RUNNER_SERVICE_TOKEN='...'
pnpm dev
```

## Health and auth checks

```bash
curl -I "$KESTREL_RUNNER_SERVICE_URL/health"
```

If you provided a custom host, port, or token, confirm the printed exports match the values your app server actually uses.

## When to use this instead of the full Deploy section

- the startup command
- the printed env exports
