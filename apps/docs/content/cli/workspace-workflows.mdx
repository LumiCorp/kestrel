Kestrel workspaces are catalog entries in `~/.kestrel/workspaces.json`.
The catalog is the durable local inventory. The current directory is only the discovery signal.

## Inspect the current workspace

```bash
kestrel workspace status
```

This resolves the current directory, registers the Git root when available, and prints the catalog binding.

## List known workspaces

```bash
kestrel workspace list
```

Inside the TUI, use:

```text
/workspace status
/workspace list
/workspace use <workspaceId|rootPath|detached>
```

## What changed

Kestrel creates catalog entries, not project-local runtime scaffolds.

Profiles are selected through the normal global profile flow.
Automation will be rebuilt on top of the catalog instead of project-local cron settings.
