The CLI and TUI are Local Core clients. They use the same Runtime, Execution
Protocol, persistence, and terminal-result contracts as Desktop and the SDK.

## Choose a reference

- [Command suite](/cli/command-suite) lists every current command-mode family.
- [CLI terminal client](/cli/kchat) explains interactive sessions and operator control.
- [Profiles, code mode, and MCP](/cli/profiles-code-mode-and-mcp) covers execution configuration.
- [Workspace workflows](/cli/workspace-workflows) covers workspace discovery and registration.
- [kcron availability](/cli/kcron) explains the implemented commands and deferred execution status.
- [Local Core HTTP bridge](/cli/runner-service) connects a trusted local server application to Local Core.

Start with `kestrel status` to inspect Local Core, or run `kestrel` to open an
interactive session.
