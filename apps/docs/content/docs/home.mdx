Kestrel gives product teams one durable execution model across local and hosted
agent experiences. The runtime owns runs, events, persistence, replay, operator
control, and evidence. Product surfaces consume those contracts instead of
copying runtime policy.

## What Kestrel Ships

- the open runtime kernel and Local Core
- a self-contained Desktop application with bundled PostgreSQL
- Kestrel One as the canonical hosted web product
- CLI and TUI workflows for local operators
- public protocol, SDK, Next.js, and observability packages
- declarative evaluations executed by released Ruhroh

Kestrel Studio is a separate private commercial product. It consumes released
public packages and does not share source with this repository.

## Ownership Model

Kestrel favors explicit boundaries: browsers call trusted app servers, app
servers call the runner service, and cross-repository products consume exact
released packages. Ruhroh owns evaluator execution. Studio owns private
authoring workflows. This repository remains the public source of truth for
runtime and product contracts.
