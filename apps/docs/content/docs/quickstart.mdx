You do not need to understand the whole Kestrel Suite before you begin. Choose the path that matches your immediate goal; each one reaches a useful result before introducing the deeper runtime concepts.

## Run agents on your computer

Choose **Kestrel Desktop** when the work begins with files or repositories on your machine. The application keeps your workspace, conversation, run history, and operator decisions together so you can close it and continue later.

[Download and install Kestrel Desktop](/desktop/install)

## Continue work with a team

Choose **Kestrel One** when several people need to continue the same agent work with shared context and clear access controls. You will need an invitation from your Kestrel One organization administrator.

[Start with your Kestrel One invitation](/kestrel-one/getting-started)

## Add Kestrel to your product

Choose **Build** when you are adding durable agent execution to an application you already own. The first tutorial creates a typed agent, runs it through the SDK, and shows the human-facing response and structured result separately.

[Build your first agent](/build/building-your-first-agent)

<Callout title="Not sure which path fits?">
  <p>Start with Desktop if you want to experience Kestrel before writing integration code. Start with Build if your immediate goal is an API or application integration.</p>
</Callout>
