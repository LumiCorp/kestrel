Archive material is not part of the main learning path, but it is still important. Many of Kestrel's sharp edges make more sense once you see the original design trade-offs or the operator runbooks that shaped them.

This section is for readers who need rationale, not current API guarantees.

## How To Use This Section

- Start with the current docs, runtime, operations, or reference sections first.
- Use archive pages to understand why a current boundary or workflow looks the way it does.
- If an archive page conflicts with a current root doc, current reference page, or current operations page, the current doc wins.

## Start here if you are...

- working on workspace-local behavior or `kcron`: start with the workspace-local profiles and scheduler plans in the plans archive
- working on the runner-service boundary: start with the docs-site and web-runner design material
- touching replay or operations flows: start with the operator runbooks and replay-adjacent plans
