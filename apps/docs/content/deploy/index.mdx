Operate covers the execution boundary behind a server-side Kestrel integration.
The main job is not "host a site." It is to keep the application-to-runtime
trust model intact while moving from a Local Core bridge in development to a
remote runner service in a hosted environment.

This section is for teams that already have the tutorial working and now need to run it in a more serious environment.

## Start here if you are...

- comparing the Local Core HTTP bridge with a remote runner service: read [Connect to a runner service](/operate/runner-service)
- wiring credentials and request context correctly: read [Environment and auth](/operate/environment-and-auth)
- treating the demo like a real product: read [Production operating model](/operate/deployment)
- debugging a broken integration: read [Deployment troubleshooting](/operate/troubleshooting)
