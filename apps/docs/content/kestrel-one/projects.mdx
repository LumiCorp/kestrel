A **Project** is the shared home for work that should outlive one conversation or one person. It groups the members, Threads, instructions, files, and selected Knowledge that belong to the same outcome.

## Create a Project

1. Open **Projects** at `/projects` and create a Project with a recognizable name and purpose.
2. Add only the members who should see its private context and Threads.
3. Add concise instructions and the files or Knowledge required for the first task.
4. Create the Thread from inside the Project.

## Understand Project roles

| Role | What the person can do |
| --- | --- |
| Owner | Manage membership, Project lifecycle, context, files, Threads, and sharing |
| Editor | Update context and files, and create or continue Threads |
| Member | Read and continue the Project Threads they can access |

Every Project retains at least one owner. Archiving removes completed work from active views while preserving its history.

<Outcome>
  <p>A teammate can enter the Project, open its Thread, and see the shared instructions and files that should guide the next turn.</p>
</Outcome>
