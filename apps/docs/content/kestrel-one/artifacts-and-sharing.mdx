An **artifact** is an output richer than a chat message: a file, structured dataset, code result, sheet, or rendered interface. Kestrel One keeps artifacts attached to the Thread and Project that produced them so their origin remains clear.

## Review an artifact

Open the artifact from its Thread, confirm that it represents the intended run, and inspect any source or context references before distributing it. Access follows the owning Thread and Project; a person who cannot open that work cannot retrieve its private artifact through another route.

## Share a transcript

Sharing publishes an anonymized, read-only transcript at a private link. It does not grant access to the organization, Project, private files, Knowledge, provider credentials, or run controls.

1. Review the transcript and artifacts for private information.
2. Use the Thread sharing action.
3. Open the public link in a signed-out browser to verify the published view.
4. Revoke the share when it should no longer be available.

<Outcome>
  <p>The recipient can read the approved public representation, while the surrounding Project and organization remain inaccessible.</p>
</Outcome>
