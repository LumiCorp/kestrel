A **Thread** is one continuing line of work. It keeps the conversation, contributors, files, artifacts, Project context, and sharing state needed to understand the result and take the next step.

## Choose the right home

- Start a standalone Thread for work that does not yet need shared Project context.
- Start from a Project when the task should use its members, instructions, files, or Knowledge.
- Archive completed Threads to preserve history without keeping them in the active list.

## Hand work to a teammate

Before handing off, leave the Thread at a clear terminal or waiting state and point to the relevant artifact. The next authorized person can read the same history, inspect the current state, and add the next turn under their own identity.

<Outcome>
  <p>The Thread shows both contributors, preserves the earlier result, and lets the next person continue from where the work stopped.</p>
</Outcome>

Moving a standalone Thread into a Project changes who may access it and which context may shape future turns. Kestrel One discloses that change before accepting it.
