Release tracks are the repo's way of grouping work into operationally meaningful slices.

## What They Contain

Each track currently defines:

- an id and title
- status and owner
- dependencies on other tracks
- required checks
- file prefixes covered by the track

## Why They Matter

They connect source areas to release discipline. Instead of treating the repo as a flat set of files, release tracks describe which checks matter most for which kind of work.
