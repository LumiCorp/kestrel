## Runtime kernel

The durable core in `src/` that owns execution stages, persistence behavior, and replayable event history.

## Runner service

The HTTP or process boundary that receives application-facing requests and executes work against the runtime.

## Scene

A deterministic scenario used for evaluation, replay, or baseline comparison.

## Replay

The ability to inspect or re-run a durable execution history and compare sequence or terminal behavior against expected outcomes.

## Workspace

A local project context that can carry Kestrel configuration and be registered for scheduler eligibility.

## Artifact

A persisted output or claim-associated record produced during execution that can be reviewed later.
