Operations is where the runtime's promises become maintainable practice. These pages cover the things a team needs once Kestrel matters in production or in a serious evaluation loop.

This section is for maintainers and operators who need incident guidance and release discipline, not only product tutorials.

## Current vs Historical

Operations pages describe the current operator posture. Use [Archive](/archive) for older migration, shutdown, and parity-program rationale when you need historical context rather than the current runbook.

## Start here if you are...

- debugging a production-shaped failure: read [Reliability](/operations/reliability)
- steering or stopping live work: read [Operator control workflows](/operations/operator-control-workflows)
- inspecting task graph, project snapshot, or project review state: read [Review and state workflows](/operations/review-and-state-workflows)
- inspecting reports, memory-linked artifacts, and replay evidence: read [Artifact inspection](/operations/artifact-inspection)
- validating declared behavior ownership: read [Evaluations with Ruhroh](/operations/evaluations)
