<GuideNote step="Release checks">
These checks protect the contracts used to plan, execute, stream, and record Kestrel work.
</GuideNote>

Every retained automated test belongs to a named contract and a required CI lane.

## Core gates

```bash
pnpm run governance:check
pnpm run test
pnpm run test-proofs:check
pnpm run ci:local -- --base origin/main
```

## What each gate protects

| Gate | Protects |
| --- | --- |
| `governance:check` | architecture, ownership, and proof-registry validity |
| selected CI lanes | executable contract counterexamples in their declared environments |
| `test-proofs:mutations` | current killed mutations for high- and critical-risk contracts |
| `ruhroh:validate` | Ruhroh configuration validity, not evaluation execution |

## When a check fails

Use the failing check to narrow the investigation:

- an unintended prompt change
- runtime regression
- a broken package or architecture rule
- replay instability

Review the first failing check and its output before changing runtime behavior.
