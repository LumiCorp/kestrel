Use the standard checks together so a passing unit test does not hide a prompt, replay, or evaluation regression.

## Core gates

```bash
pnpm run governance:check
pnpm run test
pnpm run prompt-suite
```

For runtime and determinism work:

```bash
pnpm run evals:release-check
```

## What each gate protects

| Gate | Protects |
| --- | --- |
| `governance:check` | required architecture and package rules |
| `test` | code-level behavior |
| `prompt-suite` | reference-agent prompt behavior |
| `evals:release-check` | released Ruhroh scenarios and evaluation metadata |

## When a check fails

Use the failing check to narrow the investigation:

- an unintended prompt change
- runtime regression
- a broken package or architecture rule
- replay instability

Review the first failing check and its output before changing runtime behavior.
