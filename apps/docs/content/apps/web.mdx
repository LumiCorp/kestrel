Kestrel One gives agent work a shared team workspace. A teammate can open the same Thread, see which Project context shaped it, inspect its artifacts, and move it forward without asking the original author to reconstruct everything.

## How team work stays coherent

- **Threads** keep the conversation, runtime session, authorship, files, and artifacts.
- **Projects** group related Threads with members, instructions, and files.
- **Apps** expose named capabilities through personal or shared connections.
- **Environments** provide the execution, Workspace, inference, and shared capability boundary.
- **Context revisions** record exactly which Project information shaped each new turn.
- **Knowledge** gives authorized work access to reusable organization material.
- **Sharing** publishes an approved read-only view without exposing the surrounding organization.

Kestrel One is available to invited teams. Start with [your invitation and first Thread](/kestrel-one/getting-started), then learn how [Apps](/kestrel-one/apps) and [Environments](/kestrel-one/environments) govern what agents can use.
