Kestrel Desktop helps you hand off meaningful work in a local project without losing sight of it. Use it to understand an unfamiliar repository, investigate a problem, plan a change, produce files, or review work that is already underway.

Useful work rarely fits into one prompt. It develops as the agent discovers information, makes decisions, changes files, asks for input, and sometimes encounters a problem. Desktop keeps the conversation, results, and current state together so you can leave and return without reconstructing the task.

## Give the work somewhere to live

Open a project, choose the model you want to use, and describe the outcome you need. Desktop keeps the project, conversation, active work, results, and recovery controls in one place.

### Start from a real project

Each session is attached to the local workspace where the work belongs. Files, requests, and results do not become a disconnected conversation you must later match back to the project.

### Leave and come back

Close the application, return tomorrow, and continue the same session. The earlier conversation, completed runs, and current project remain available, so you do not have to reconstruct the task before moving forward.

### Know where things stand

See whether work is active, completed, waiting for you, cancelled, or failed. Review the answer, generated artifacts, recent activity, and recorded outcome instead of relying on the last visible message to explain everything that happened.

### Step in when the work needs you

Inspect, steer, stop, resume, retry, or recover work from the same session. Your intervention becomes part of its history rather than forcing you to discard the run and begin again with a new prompt.

## What people use Desktop for

Desktop is most useful when the outcome matters beyond the current conversation. Common examples include:

- investigating a bug and keeping the findings with the project
- planning and carrying out a change that touches several files
- returning to unfinished work without explaining the project again
- reviewing what changed, what was produced, and what still needs attention
- correcting course when a task waits, fails, or heads in the wrong direction
- turning a successful manual task into repeatable scheduled work

The goal is simple: help you delegate substantial work while preserving the context and control needed to trust the result.
