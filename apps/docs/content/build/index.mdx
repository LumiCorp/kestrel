The Build journey follows one Workspace Copilot from a first server-side request to streaming, memory, waiting, Next.js routes, and observability. You keep the same agent and session model throughout, so every step extends something already working.

## Start building

1. [Build your first agent](/build/building-your-first-agent) and inspect its terminal result.
2. [Stream the next request](/build/running-your-first-streamed-request) without changing session continuity.
3. Add [explicit session memory](/build/adding-session-memory).
4. Expose the agent through [Next.js routes](/build/integrating-with-nextjs).
5. Handle [waiting, resume, and cancellation](/build/waiting-resume-and-cancellation).
6. Add [observability](/build/adding-observability) before production operation.

The [Workspace Copilot demo](/build/workspace-copilot-demo) shows the completed example and its extension points.

## Handle results correctly

Before shipping, read [Protocol and results](/build/protocol-and-results). A terminal result always contains `assistantText: string | null`; human-facing text is not inferred from structured finalized data. Use [Runner events](/build/runner-events) for the exact streaming vocabulary and [Upgrading to 0.6](/build/upgrading-to-0-6) for compatibility changes.
