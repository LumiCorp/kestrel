Add a subscription for activity that should remain observable after a browser request ends.

Use `stream()` while someone is waiting for one request. Use `subscribe()` when another part of your application needs selected updates from a session, run, or thread.

## Observe work outside one request

Workspace Copilot eventually needs two kinds of updates:

- the end user wants live output while an interactive request is running
- a background process needs important session updates after the original request has ended

Use a subscription for the second case so background monitoring does not depend on the original browser connection.

## Open a filtered subscription

```ts
const events = workspaceCopilotAgent.subscribe(
  {
    sessionId: "workspace-copilot-session-001",
    eventTypes: ["task.updated"],
  },
  workspaceCopilotContext,
);

for await (const event of events) {
  console.log(event.type, event.sessionId);
}
```

Every subscription needs a session, run, thread, or event filter. The root SDK does not provide an unfiltered global event stream.

## When to scope the filter

| Scope | Use it when |
| --- | --- |
| `sessionId` | you want the whole Workspace Copilot conversation or job lifecycle |
| `runId` | you care about one specific run |
| `threadId` | you are watching one thread lineage in a larger session |
| `eventTypes` | you only need particular background updates |

## Request stream vs background subscription

| Method | Best for | Avoid using it for |
| --- | --- | --- |
| `stream()` | one active request | long-lived session monitoring |
| `subscribe()` | filtered background observation | replacing user-facing request streaming |

## Use streams and subscriptions together

- Workspace Copilot now has a request stream and a separate background subscription
- an app can watch session-level changes without binding everything to one browser request
- monitoring tools can watch `workspace-copilot` without holding the original request open

## Why the separation helps

Once you add automation and scheduled reports, important work can continue after a user leaves the page. Subscriptions let your application observe that work without misusing a request stream.
