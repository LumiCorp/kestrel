Kestrel Desktop packages the local runtime and its managed database into one application. You will not need to install PostgreSQL or run services from a source checkout.

<DesktopDownload />

## Beta availability

The Desktop `0.6.0-beta.0` artifact is not published yet. The releases page currently contains earlier builds, which do not match these guides. A direct 0.6 download will appear here when the artifact is available.

When the 0.6 Beta is published:

1. Download the macOS artifact from the release page.
2. Confirm that the version is `0.6.0-beta.0` and read the signing or notarization notes.
3. Move Kestrel to your Applications folder.
4. Open Kestrel and approve only the operating-system prompts described by the release notes.

<Outcome>
  <p>After installing the 0.6 Beta, Kestrel opens to provider setup. You should not be asked to configure a database, runner service, or repository checkout.</p>
</Outcome>

## If the application does not open

Return to the release notes and confirm that you downloaded the correct platform artifact. Do not disable operating-system security controls globally. If the documented platform-specific flow still fails, capture the warning and report it through the release issue tracker.
