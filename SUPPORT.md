# Kestrel Support

Use the path below that matches the problem. Including the right identifiers
and evidence helps the issue reach the component that owns it.

## Get Help

| Need | Where to go |
| --- | --- |
| Reproducible bug | [Open a bug report](https://github.com/LumiCorp/kestrel/issues/new?template=bug_report.md) |
| Feature or product request | [Open a feature request](https://github.com/LumiCorp/kestrel/issues/new?template=feature_request.md) |
| Existing report or workaround | [Search issues](https://github.com/LumiCorp/kestrel/issues) |
| Setup or product guidance | [Kestrel documentation](https://docs.kestrelagents.dev) |
| Suspected vulnerability | Follow the private process in [Security](SECURITY.md) |

If GitHub Discussions is enabled for the repository, use it for open-ended
usage questions. Otherwise, open an issue and label the request clearly as a
question.

## Include With a Bug Report

- Kestrel version, commit, or release artifact
- product surface: Desktop, Kestrel One, CLI/TUI, runner service, SDK, or other
  package
- operating system, Node.js version, and installation method when relevant
- exact command or user action
- expected and observed behavior
- run, session, Thread, or Project identifier when safe to share
- terminal state and the smallest useful event or error excerpt
- minimal reproduction and whether it repeats

Do not paste provider keys, runner tokens, signing material, private prompts,
customer data, or unrestricted support bundles into a public issue.

## Before You File

1. Check the [0.6 Beta release status](apps/docs/content/start/release-status.mdx)
   and [compatibility guide](apps/docs/content/reference/compatibility.mdx).
2. Search [existing issues](https://github.com/LumiCorp/kestrel/issues).
3. Use the owning troubleshooting guide:
   [Desktop](apps/docs/content/desktop/troubleshooting.mdx) or
   [deployment](apps/docs/content/deploy/deployment-troubleshooting.mdx).
4. Preserve logs, artifacts, checkpoints, and identifiers before resetting or
   deleting state.

## Security

Do not report vulnerabilities in public issues. Use
[GitHub Security Advisories](https://github.com/LumiCorp/kestrel/security/advisories/new)
and follow [SECURITY.md](SECURITY.md).
