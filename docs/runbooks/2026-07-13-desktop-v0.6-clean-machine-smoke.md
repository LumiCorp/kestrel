# Kestrel Desktop 0.6 Clean-Machine Smoke

Run this checklist on a clean macOS user profile from the notarized ZIP, outside the contributor checkout. Kestrel Local Core is the shared local source of truth for Desktop and CLI. Preserve screenshots, the Desktop support bundle, and command output for every failed item.

## Preflight

- [ ] Record macOS version, hardware architecture, Git commit, artifact SHA-256, and artifact filename.
- [ ] Confirm `xcrun stapler validate Kestrel.app` succeeds after extraction.
- [ ] Confirm `spctl --assess --type execute --verbose=4 Kestrel.app` succeeds.
- [ ] Confirm no Kestrel, Electron, Local Core, Postgres, or runner process from an earlier smoke is active.
- [ ] Confirm the first test profile has no Kestrel data.

## Clean First Launch

- [ ] Launch from Finder without a Privacy & Security override.
- [ ] Confirm Gatekeeper identifies the app and does not show “Apple could not verify.”
- [ ] Confirm Desktop reports version `0.6.0`.
- [ ] Confirm Local Core becomes healthy with database mode `pglite` or `PGlite`.
- [ ] Confirm no bundled `postgres-bundle` exists in `Kestrel.app/Contents/Resources`.
- [ ] Confirm no independent `cli/runner/main.ts` process is launched by Desktop.
- [ ] Complete provider selection and add one project.
- [ ] Run a prompt, observe streamed events, restart Desktop, and confirm the thread and run history replay.

## Safe Recovery

- [ ] Induce a recoverable Core startup failure in an isolated test profile.
- [ ] Confirm the highest-severity blocked item supplies the screen summary and first quick action.
- [ ] Confirm the database action is `Retry Database`, not `Repair Database`.
- [ ] Click the action repeatedly and confirm no database or state directory is deleted or replaced.
- [ ] Capture a support bundle and confirm the root error is present.

## Shared-Core Ordering

- [ ] Desktop-first then CLI: start Desktop, run one turn, attach the CLI, and confirm both clients see the same Core state.
- [ ] CLI-first then Desktop: start the CLI, run one turn, launch Desktop, and confirm Desktop attaches without a second authority.
- [ ] concurrent launch: start both clients together and confirm one Local Core authority, one PGlite store, and no independent runner processes.
- [ ] stale lock: terminate the owning client, relaunch, and confirm Core recovers only a demonstrably stale lock.
- [ ] inherited `DATABASE_URL`: launch with an unrelated inherited value and confirm default mode still uses PGlite.

## 0.5 Coexistence and Rollback

- [ ] Use a copy of a real 0.5.1 profile containing the documented state bridge.
- [ ] Launch 0.6 and confirm imported UI state appears while the 0.5 state remains untouched.
- [ ] Quit 0.6, relaunch the 0.5.1 recovery artifact, and confirm its original state still works.
- [ ] Relaunch 0.6 and confirm the isolated 0.6 state remains consistent.

## Exit Criteria

The smoke passes only when every item above is complete, the packaged GUI smoke passes, no orphaned Desktop/Core/runner process remains, and the evidence identifies the exact notarized artifact tested. A prepared checklist is not execution evidence.
