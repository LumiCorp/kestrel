# Kestrel Desktop 0.6 macOS Release

Kestrel Desktop 0.6 is a macOS-first direct-distribution release. Kestrel Local Core is the sole local authority for execution and durable product state. Desktop connects to Core's authenticated execution protocol; it does not launch a separate runner or Postgres server.

## Release Contract

- Version: `0.6.0` across the suite manifests.
- Default store: embedded PGlite under the isolated 0.6 state epoch.
- Advanced store: explicit external Postgres only.
- Upgrade boundary: 0.5 state remains untouched. The 0.5.1 bridge document is import input, not a database migration target.
- Distribution: Developer ID Application signed, hardened, notarized by Apple, and stapled.
- Update service: out of scope for 0.6.0; installation is a deliberate artifact replacement.

An ad-hoc-signed artifact is a developer build, not a release candidate and not a user-distributable 0.6 artifact.

## Credential Setup

Install a valid `Developer ID Application` identity in the build user's keychain. Store Apple notarization credentials in a named keychain profile rather than putting secrets in the repository or command line:

```bash
xcrun notarytool store-credentials "kestrel-notary"
security find-identity -v -p codesigning
```

The build requires the identity name and the keychain profile name. It never requires the underlying password as an environment variable.

## Build and Notarize

From the repository root:

```bash
CI=true pnpm run governance:check
CI=true pnpm run test
CI=true pnpm run prompt-suite
CI=true pnpm run evals:release-check

KESTREL_DESKTOP_RELEASE=1 \
KESTREL_DESKTOP_SIGN_IDENTITY="Developer ID Application: YOUR NAME (TEAMID)" \
KESTREL_DESKTOP_NOTARY_PROFILE="kestrel-notary" \
pnpm run desktop:package

KESTREL_DESKTOP_RELEASE=1 pnpm run desktop:release-check
```

Release packaging fails if the signing identity or notary profile is absent. It signs with hardened runtime, submits a temporary ZIP with `notarytool --wait`, staples and validates the app ticket, runs a Gatekeeper assessment, and only then creates `apps/desktop/out/Kestrel-0.6.0-darwin-arm64.zip`.

## Required Evidence

Record these values with the release candidate:

- Git commit and branch.
- Artifact filename, size, and SHA-256.
- `codesign -dv --verbose=4` authority and runtime flags.
- `xcrun stapler validate` result.
- `spctl --assess --type execute --verbose=4` result.
- Clean-machine smoke evidence from the linked checklist.

Do not mark this runbook verified or describe 0.6 as available until the notarized artifact and the clean-machine evidence both exist.
