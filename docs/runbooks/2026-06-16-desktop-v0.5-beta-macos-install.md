# Kestrel Desktop v0.5.0-beta.0 macOS Install Notes

See also: [clean-machine smoke checklist](./2026-06-16-desktop-v0.5-beta-clean-machine-smoke.md) and [Desktop README](../../apps/desktop/README.md).

## Artifact Status

Kestrel Desktop `0.5.0-beta.0` is a macOS-first unsigned beta artifact.

- Supported proof target: macOS clean-machine or clean-user-profile smoke.
- Distribution state: unsigned beta package.
- Not included: notarization, auto-update, polished Windows/Linux installer promise.
- Final release evidence requires a real clean-machine run outside the contributor checkout.
- Shared Kestrel Local Core is required before Desktop plus CLI clean-machine smoke can pass.

If macOS blocks the unsigned app on first open, use the standard macOS override flow for an app you trust: open from Finder, approve the warning in System Settings if prompted, then launch again. Do not describe this artifact as signed or notarized.

## First-Run Path

The intended first-run path is:

1. Launch Kestrel Desktop.
2. Start or connect to Kestrel Local Core.
3. Complete guided setup.
4. Choose a model provider.
5. Add provider credentials or confirm local-provider readiness.
6. Choose a local project folder.
7. Start the first Desktop session.

Setup is for provider choice, credential or local-provider readiness, and project selection. Recovery is for runtime, database, or packaged-resource failures. Settings is durable configuration after or during setup. Diagnostics is logs and support bundle access. Advanced is low-level service and environment configuration.

Kestrel Local Core is the shared local source of truth. Desktop is the primary shell over that Core. The CLI/TUI and beta `kcron` use the same Core in packaged default mode rather than creating separate product state.

## Provider Paths

### OpenRouter

OpenRouter is the hosted-provider smoke path for this beta.

- Choose `OpenRouter` during setup.
- Enter `OPENROUTER_API_KEY`.
- Keep the default model unless the smoke run explicitly needs another model.
- Confirm the first session can start after project selection.

The key is stored locally in Desktop settings for Desktop runs. It should be redacted from support bundles and diagnostics.

### Ollama

Ollama is included as a beta local-provider path.

- Install and start Ollama before the local-provider smoke.
- Choose `Ollama` during setup.
- Confirm Desktop does not require an API key for this provider.
- Use the default local base URL unless the local Ollama server uses a custom port.

If no model is available locally, the expected beta behavior is a clear local-provider readiness or model failure, not a generic Desktop crash.

## Beta Labels

These surfaces are allowed in the `0.5.0-beta.0` artifact but are not stable public promises:

- local providers: Ollama and LM Studio
- code/dev-shell workflows
- current Core-managed packaged Postgres path until clean-machine proof is captured
- `kcron` local automation
- unsigned packaged Desktop artifacts

## Build And Release Checks

Before handing the artifact to a clean-machine tester, run:

```bash
pnpm --filter @kestrel/desktop prepare:resources
pnpm run desktop:release-check
pnpm --filter @kestrel/desktop test
pnpm run desktop:package
```

`desktop:release-check` verifies suite version drift, Desktop app version sourcing, required release docs, required packaged runtime resources, excluded local `.env` files, and staged dependency presence when the package stage exists.

The release check also requires the Local Core contract resources and shared-Core smoke/migration runbooks. It is expected to fail before public beta handoff if Desktop or CLI defaults drift back to shell-specific product state.

## Known Limitations

- This beta is macOS-first and unsigned.
- The final clean-machine smoke must be run outside the repo checkout.
- Auto-update is out of scope.
- Windows and Linux are not official v0.5 beta proof targets unless separately tested.
- Local-provider failures may require local server/model setup outside Kestrel.
- External database mode is Advanced and requires a valid `DATABASE_URL`; missing URL is a configuration blocker, not a Desktop crash.
- Packaged default mode must not silently use an inherited `DATABASE_URL` from the host terminal.
