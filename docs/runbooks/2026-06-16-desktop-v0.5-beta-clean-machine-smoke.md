# Kestrel Desktop And CLI v0.5.0-beta.0 Clean-Machine Smoke Checklist

See also: [macOS install notes](./2026-06-16-desktop-v0.5-beta-macos-install.md), [Local Core migration evidence](./2026-06-17-local-core-beta-migration-evidence.md), and [Desktop first-run capability audit](../analysis/2026-06-03-packaged-desktop-first-run-capability-audit.md).

## Required Evidence

Capture the result, screenshots when useful, and the generated support bundle or relevant log snippets for each failure.

- macOS version and chip architecture.
- Artifact name and version shown by Desktop.
- Whether the user profile had prior Kestrel Desktop, CLI, or Kestrel Local Core data.
- Kestrel Local Core home, source, version, schema version, lock state, and database mode.
- OpenRouter first-run result.
- Ollama first-run result.
- Workspace path used for smoke.
- Desktop-first then CLI shared-state result.
- CLI-first then Desktop shared-state result.
- Any startup recovery screen, diagnostic action, or support bundle generated.

## Preflight

- [ ] Use a real clean macOS machine or a clean macOS user profile.
- [ ] Ensure no prior Kestrel Desktop, CLI, or Kestrel Local Core data is present for the first launch.
- [ ] Keep the repo checkout closed and do not run from `pnpm run desktop:dev`.
- [ ] Have an OpenRouter API key available.
- [ ] Install and start Ollama for the local-provider smoke.
- [ ] Prepare a small local project folder that can be safely used for the first session.
- [ ] Extract the CLI beta tarball on the same profile without installing source-backed checkout shims.
- [ ] Run `node --import tsx scripts/local-core-legacy-state.ts --json` on the packaging machine before handoff, and record whether the detector reports old Desktop or CLI state.

## Local Core Baseline Smoke

- [ ] Launch Kestrel Desktop from the packaged beta artifact.
- [ ] Open diagnostics or status and capture the Kestrel Local Core home.
- [ ] Capture `core/api.sock`, `core/api.token`, and `core/lock.json` existence under the Core home without printing the token value.
- [ ] Confirm `core/lock.json` records one daemon owner pid, daemon executable, Core version, schema version, API socket, database socket, and heartbeat.
- [ ] Confirm the Core home is the shared product root, not Desktop `@kestrel/desktop` user data.
- [ ] Confirm the default database mode is Core-managed local state or an explicit Advanced external database mode.
- [ ] Confirm no step uses host Postgres, Docker, Homebrew Postgres, repo `.env`, or a repo checkout path.

Expected result: Desktop reports Kestrel Local Core readiness and uses shared local Kestrel data rather than Desktop-private product state.

## OpenRouter Hosted-Provider Smoke

- [ ] Launch Kestrel Desktop from the packaged beta artifact.
- [ ] Confirm guided setup opens automatically.
- [ ] Choose `OpenRouter`.
- [ ] Enter `OPENROUTER_API_KEY`.
- [ ] Choose the local project folder.
- [ ] Finish setup.
- [ ] Start a first Desktop session.
- [ ] Confirm the session reaches a useful assistant response or a clear model/provider error.
- [ ] Restart Desktop.
- [ ] Confirm provider and project settings persist.

Expected result: Desktop reaches a usable workspace without contributor setup docs, and any model/provider failure is specific enough to act on.

## Ollama Local-Provider Smoke

- [ ] Reset Desktop user data or use another clean profile.
- [ ] Start Ollama locally.
- [ ] Launch Kestrel Desktop from the packaged beta artifact.
- [ ] Choose `Ollama`.
- [ ] Confirm setup does not ask for an API key.
- [ ] Confirm the base URL is local and understandable.
- [ ] Choose the local project folder.
- [ ] Finish setup.
- [ ] Start a first Desktop session.

Expected result: Desktop either uses the local Ollama model or surfaces a local-provider readiness/model issue without crashing. This path is beta.

## Recovery And Diagnostics Smoke

- [ ] Switch to external database mode without `DATABASE_URL`.
- [ ] Restart runtime or Desktop.
- [ ] Confirm recovery/readiness explains the missing database URL.
- [ ] Generate a support bundle.
- [ ] Confirm provider keys and database URL-like secrets are redacted.
- [ ] Confirm logs identify runtime/database/resource state.

Expected result: configuration blockers are explained as recovery states, not generic crashes.

## Shared Shell Smoke

### Desktop-first then CLI

- [ ] Start from a fresh profile.
- [ ] Launch Desktop and complete setup.
- [ ] Create or select a workspace and start a session.
- [ ] Run the packaged `bin/kestrel status`.
- [ ] Confirm CLI reports the same Kestrel Local Core home, version, schema version, and database mode as Desktop.
- [ ] Confirm CLI reports the same Local Core API socket and daemon owner as Desktop diagnostics.
- [ ] Run a CLI command that lists or inspects the selected workspace/session.
- [ ] Confirm CLI sees the Desktop-created state without `KESTREL_HOME`, source checkout shims, repo `.env`, or manual `DATABASE_URL`.

Expected result: Desktop-first then CLI uses the same Local Core source of truth.

### CLI-first then Desktop

- [ ] Reset to a fresh profile.
- [ ] Start with the packaged CLI tarball.
- [ ] Run `bin/kestrel status` and capture the Kestrel Local Core home.
- [ ] Register or select a workspace through CLI.
- [ ] Launch Desktop.
- [ ] Confirm Desktop reports the same Core home and can see the CLI-selected workspace/session state.
- [ ] Confirm Desktop attaches to the existing Core daemon instead of starting a second owner.

Expected result: CLI-first then Desktop uses the same Local Core source of truth.

### Concurrent Launch

- [ ] Reset to a fresh profile.
- [ ] Start Desktop and `bin/kestrel status` at roughly the same time.
- [ ] Confirm only one Local Core owner/migration path wins.
- [ ] Confirm both shells use the same `core/api.sock` and token-backed Local Core API.
- [ ] Confirm the losing shell waits, attaches, or reports a clear Core readiness state.

Expected result: concurrent launch does not create separate Core homes, separate databases, or competing migrations.

### Stale Lock Repair

- [ ] Create the approved stale lock fixture or kill the Core owner process after recording the lock state.
- [ ] Launch Desktop and run `bin/kestrel status`.
- [ ] Confirm stale lock is reported as stale lock repair, not as generic database or runtime failure.
- [ ] Confirm repair does not move or overwrite old Desktop or CLI state.

Expected result: stale lock recovery is explicit and shared by Desktop and CLI diagnostics.

### Inherited `DATABASE_URL` Rejection

- [ ] In a terminal, export an unrelated `DATABASE_URL` pointing at a fake or host Postgres endpoint.
- [ ] Run packaged `bin/kestrel status`.
- [ ] Launch Desktop from Finder or a clean shell.
- [ ] Confirm packaged default mode still uses Kestrel Local Core state unless Advanced external database mode is explicitly selected.

Expected result: inherited `DATABASE_URL` is ignored or rejected in packaged default mode and cannot silently redirect the product source of truth.

## Pass Criteria

The macOS beta smoke passes when:

- Desktop opens from the packaged artifact.
- Desktop and CLI report the same Kestrel Local Core home in default packaged mode.
- Guided setup works for OpenRouter.
- Ollama setup confirms no API key is required and fails gracefully if local model setup is incomplete.
- Project selection and restart persistence work.
- Recovery and diagnostics produce actionable evidence.
- No step requires a contributor checkout, local `.env`, `pnpm`, host Postgres, Docker, Homebrew Postgres, or inherited `DATABASE_URL`.
- `pnpm run local-core:release-smoke` has passed before manual clean-machine UI evidence is collected.
